Main capabilities:

comparison between venues

2. VENUE WORKSPACE SHELL
A. Capability summary

This is the operational spine of the venue-level app.

Current subnav sequence:

Overview
Assessment
Signals
Console
Plan
Report
History
B. What value must survive

The value is not the tabs themselves.
The value is that the workspace expresses a real operational sequence.

This shell says:

start with context
assess reality
review machine interpretation
think and discuss
formalize diagnosis
execute
preserve continuity

That is one of the strongest structural achievements in OIS_Cafe_v2.

C. Target home in VOIS

Best future home:

Manager venue shell
with read-only/filtered owner access where appropriate
pocket should not inherit this whole shell directly
D. Core vs ontology
Core

Everything here is core:

workspace shell
subnav
page ownership
venue identity persistence
navigation continuity
role-aware access to tabs
Ontology

Only page labels or content hints if they vary by domain

This page is where the machine becomes accountable.

Without this page, the system becomes much more opaque and less credible.
