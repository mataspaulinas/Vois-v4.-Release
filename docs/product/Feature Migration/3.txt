6. CONSOLE / COPILOT
A. Capability summary

This is not a simple assistant chat.

It is a multi-state, context-rich operational reasoning surface with:

persisted threads
thread groups
archived threads
cross-venue thread access
attachments
file-aware interaction
signal update proposals
proactive prompts
conversation continuity

From the structure, it behaves more like:
an operational conversation workspace
than
a generic AI panel.

B. What value must survive

The most important value here is not “chat.”

The value is:

persistent reasoning over time
contextual operational dialogue
ability to attach files and discuss them
thread continuity across venue work
actionable assistant interventions
signal correction proposals inside conversational flow
workspace memory and human review

This is a very major capability.

C. User jobs this solves

It helps the user:

ask context-aware questions
continue a diagnostic thread over time
inspect uploaded material
get prompted proactively
keep discussions organized by thread
archive old lines of inquiry
move between venue-local and broader context
apply AI-suggested signal changes with review

7. REPORT
A. Capability summary

The Report page is the formal artifact layer.

This is where the app turns assessment + reviewed signals into:

structured output
persistent narrative
a document-like operational result

It includes saved artifact behavior and likely report toolbar actions.

B. What value must survive

The report is not just a view.

Its essential value is:

formalized diagnostic output
saved artifact
reviewable historical object
bridge between diagnosis and execution
narrative explanation of what the system sees

Without this, the system feels like it only computes but does not formalize.

C. User jobs this solves

It helps the user:

understand the current diagnosis clearly
review a structured narrative
preserve the result of an assessment cycle
possibly export or share in the future
compare report evolution over time
D. Feature inventory

Observed or strongly implied:

report rendering
report toolbar
save report HTML
report retrieval by filename/artifact
report persistence by venue
likely connection to plan and assessments
E. Product value

High.

This is one of the main things that makes the system feel professional rather than ephemeral.

F. Core vs ontology
Core
report object
artifact persistence
report rendering shell
toolbar
history linkage
report provenance
retrieval by run/history
Ontology
domain language blocks
explanatory wording
domain framing of findings
domain-specific report sections where appropriate
Split rule

The report engine/artifact model is core.
The specific meaning and explanatory content is ontology-grounded.

G. Migration decision

MIGRATE / ALIGN

The capability absolutely should survive, but it must align with the newer report spine and trust-artifact model.

H. Priority

Tranche 2

Important, but can follow once:

assessment
signals review
plan direction
are stable
I. Risk

Medium

Main risks:

duplicate report truth
unclear relationship to reviewed signals
unclear relationship to active/draft plan
loss of provenance
report feeling detached from history
J. What devs must preserve
persisted report artifact
formal narrative structure
clear relationship to the underlying assessment cycle
historical retrievability
toolbar/action surface
K. What devs must not carry over
detached report rendering with weak provenance
report as a free-floating HTML blob with no lineage

report generation that bypasses reviewed signal truth
L. Recommended VOIS implementation shape

The report in VOIS should likely include:

Primary zones
report summary
findings / themes
supporting signal references
recommended intervention framing
provenance / trust metadata
link to source assessment and source reviewed signals

Strong rule

The report must clearly show what run / assessment / signal state it came from.

M. Acceptance criteria

This migration is successful when:

reports are saved as real artifacts
users can reopen them later
report content is tied to a specific assessment/run lineage
reviewed signal truth is visible or traceable
report and plan relationship is understandable
the page feels formal and trustworthy, not disposable
